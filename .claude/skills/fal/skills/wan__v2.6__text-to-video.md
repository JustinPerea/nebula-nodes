# Wan v2.6 Text-to-Video

## Overview
- **Slug:** `wan/v2.6/text-to-video`
- **Category:** Text-to-Video
- **Creator:** [Alibaba Cloud (DashScope)](https://www.alibabacloud.com/help/en/model-studio/text-to-video-api-reference)
- **Best for:** Cinematic multi-shot storytelling with high temporal consistency and optional background audio.
- **FAL docs:** [fal.ai/models/wan/v2.6/text-to-video](https://fal.ai/models/wan/v2.6/text-to-video)
- **Original source:** [Alibaba Cloud DashScope](https://www.alibabacloud.com/help/en/model-studio/text-to-video-api-reference)

## What it does
Wan v2.6 is a state-of-the-art video generation model developed by Alibaba that transforms text prompts into high-definition (up to 1080p) videos. It is specifically designed for **multi-shot narratives**, allowing users to specify scene changes and timing within a single prompt. The model excels at maintaining character and environmental consistency across these transitions and supports native audio-visual synchronization by allowing users to provide a background audio track during generation.

## When to use this model
- **Use when:** You need a coherent story with multiple camera angles or scenes in one clip; you require high-resolution (1080p) output for professional social media or marketing; you want to synchronize video motion with a specific audio track.
- **Don't use when:** You need ultra-fast "flash" generation (use `wan/v2.6/text-to-video/flash` instead); you require durations longer than 15 seconds; you need 4K resolution (not supported on FAL).
- **Alternatives:**
    - **Kling 3.0:** Better for extreme photorealism and longer durations but often more expensive.
    - **Luma Dream Machine:** Great for fluid physics and complex movement, but less structured for multi-shot timing.
    - **HunyuanVideo:** Strong open-weights alternative with good motion, but lacks the native multi-shot "director" logic of Wan 2.6.

## API — via FAL.ai
**Endpoint:** `https://fal.run/wan/v2.6/text-to-video` (sync) / `https://queue.fal.run/wan/v2.6/text-to-video` (queue)

### Input parameters
| Parameter | Type | Default | Range/Values | Description |
|---|---|---|---|---|
| `prompt` | string | (required) | max 800 chars | The text description of the video. Supports English and Chinese. Use timing tags like `[0-3s]` for multi-shot control. |
| `audio_url` | string | null | URL | Publicly accessible URL for background music/audio (WAV/MP3, 3-30s, <15MB). |
| `aspect_ratio` | enum | `16:9` | `16:9`, `9:16`, `1:1`, `4:3`, `3:4` | The visual format of the output video. |
| `resolution` | enum | `720p` | `720p`, `1080p` | Output quality tier. Note: 1080p costs more. |
| `duration` | enum | `5` | `5`, `10`, `15` | Total length of the generated video in seconds. |
| `negative_prompt` | string | `""` | max 500 chars | Elements to exclude (e.g., "watermark, text, low quality"). |
| `enable_prompt_expansion` | boolean | `true` | true/false | Uses an LLM to enrich short prompts for better visual results. |
| `multi_shots` | boolean | `true` | true/false | Enables intelligent scene segmentation. Only works if `enable_prompt_expansion` is true. |
| `seed` | integer | null | Any integer | For reproducible results. |
| `enable_safety_checker` | boolean | `true` | true/false | Filters out sensitive or non-compliant content. |

### Output
The output returns a JSON object containing:
- **`video`**: A `VideoFile` object with `url`, `width`, `height`, `fps`, and `duration`.
- **`seed`**: The integer seed used for this generation.
- **`actual_prompt`**: The expanded prompt generated by the internal LLM (if expansion was enabled).

### Example request
```json
{
  "prompt": "Shot 1 [0-5s]: A futuristic cyberpunk city at night with neon lights reflecting in puddles. Shot 2 [5-10s]: A close-up of a robotic eye scanning the environment with data overlays.",
  "resolution": "1080p",
  "duration": 10,
  "aspect_ratio": "16:9",
  "enable_prompt_expansion": true
}
```

### Pricing
- **720p:** $0.10 per second of video generated.
- **1080p:** $0.15 per second of video generated.
- *Example:* A 10-second 1080p video costs $1.50.

## Prompting best practices
- **Structured Narratives:** Use the formula `[Timing] Scene Description + Camera Angle + Lighting`. Example: `[0-4s] Low angle shot of a dragon taking flight, golden hour lighting.`
- **Multi-shot Logic:** Wan 2.6 is unique in its "Director" capability. Clearly label shots (Shot 1, Shot 2) and give each a distinct visual goal while keeping the subject the same to maintain consistency.
- **Lighting & Texture:** Keywords like "cinematic lighting", "8k resolution", "volumetric fog", and "subtle film grain" significantly enhance the aesthetic.
- **Bilingual Advantage:** If you are comfortable with Chinese, prompts in Chinese can sometimes yield slightly better nuances for traditional cultural themes due to the model's training origin.
- **Avoid Over-crowding:** While it supports multi-shot, don't try to cram 10 shots into 10 seconds. Aim for 3-5 seconds per shot for best coherence.

## Parameter tuning guide
- **Resolution (720p vs 1080p):** Use 720p for rapid prototyping. Switch to 1080p only for final renders, as it doubles the cost and increases generation time.
- **Prompt Expansion:** Leave this **ON** for simple prompts (e.g., "A cat in space"). Turn it **OFF** if you have a highly technical, professional shot-list that you don't want the AI to modify.
- **Multi-Shots:** Always keep this **ON** if your prompt contains multiple scenes or time-segments. If you want one long continuous camera movement, turn it **OFF**.
- **Audio Integration:** The model attempts to sync motion intensity with the audio rhythm. Upload high-energy audio for action scenes and ambient tracks for slow-panning landscapes.

    - `fal-ai/kling-video/v1.5/upscale`: Use Wan for the base story, then upscale the result for even higher fidelity.

## Notes & gotchas
- **Consistency:** While excellent, rapid scene changes (e.g., from a desert to a snowy mountain) can sometimes cause the subject's clothing or appearance to drift slightly.
- **Safety Checker:** The safety checker is strict; prompts involving public figures or suggestive content may be blocked even if they are relatively benign.
- **File Limits:** `audio_url` files must be under 15MB. Larger files will result in a request error.
